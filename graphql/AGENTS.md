# Agent guide — graphql/

Wires a gqlgen `*handler.Server` as an fx dependency: adds all standard
transports + extensions and mounts the endpoint + GraphiQL playground on the
shared `chi.Router`. This is the **server** side; for consuming external APIs
see `graphql/client/`.

## Wiring

```go
fx.New(
    golusoris.Core,
    httpx_router.Module,
    httpx_server.Module,
    graphql.Module, // provides *handler.Server, mounts routes
    fx.Provide(func() gql.ExecutableSchema { return generated.NewExecutableSchema(...) }),
)
```

**Provides** `*handler.Server`. **Requires** a `graphql.ExecutableSchema`
(generated by gqlgen), `chi.Router`, and `*config.Config` in the graph.
`Handler(srv)` returns an `http.Handler` for mounting outside chi.

## Config

Keys live under the `graphql` prefix (env `APP_GRAPHQL_*`):

```yaml
graphql:
  path: /graphql                      # endpoint path
  playground: true                    # enable GraphiQL UI
  playground_path: /graphql/playground
  complexity_limit: 1000              # 0 disables the complexity guard
  apq_cache: 100                      # APQ LRU size; 0 disables APQ
```

## Notes

- Transports added in order WS → SSE → OPTIONS → GET → POST → MultipartForm
  (first match wins). Introspection is always on.
- `complexity_limit: 0` and `apq_cache: 0` remove those guards — keep the
  complexity limit set on public endpoints to bound query cost.
- `playground` is a koanf bool whose zero value is `false`; `loadConfig`
  re-applies the `true` default only when no graphql keys were set.
