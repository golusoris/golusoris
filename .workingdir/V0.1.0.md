## Security & supply-chain posture

- **CI**: 7 workflows (ci, ci-go reusable, codeql, scorecard,
  release-go, rebuild-on-base, auto-assign).
- **Pinned actions**: all `uses:` in CI pin to SHAs (gosec to
  `v2.25.0` after recent Scorecard fix).
- **gosec**: extensive per-path exclusion set for legitimate dual-use
  code (weak crypto for HIBP k-anonymity, etc.). All other findings
  carry inline `#nosec` with justification.
- **CodeQL**: advanced setup (`security-extended` + `security-and-quality`
  query packs). Default setup disabled to avoid duplicate runs.
- **CSRF**: `filippo.io/csrf/gorilla` (same-origin enforcement, matches
  Go 1.25's net/http.CrossOriginProtection) — closes
  GHSA-82ff-hg59-8x73 with no upstream patch available.
- **SBOM + SLSA**: release workflow emits CycloneDX SBOM + cosign
  signatures + SLSA-L3 provenance (wired in `release-go.yml`).
- **Threat model references** in [PLAN.md §2.5](PLAN.md): SLSA L3,
  OWASP ASVS L2, NIST SSDF, EU CRA, NIS2, BSI C5, UK NCSC, ENISA,
  GDPR, EU AI Act. Framework ships scaffolding; apps assert
  compliance in their own `SECURITY.md`.

## Pre-tag checklist (run once)

- [ ] `git pull --rebase origin main && git status` — clean working tree.
- [ ] Confirm latest CI run on `main` is green (CI, CodeQL, Scorecard).
- [ ] Dependabot: 0 open alerts (re-scan cleared — as of `42d01bc`).
- [ ] [CHANGELOG.md](../CHANGELOG.md) `[0.1.0] — 2026-04-14` section
      reviewed. For v0.1.0 "Migration notes" is blank (first release);
      "Dependencies bumped" is N/A (no prior go.mod to diff against).
- [ ] Create annotated tag: `git tag -a v0.1.0 -m "v0.1.0 — first tagged release"`.
- [ ] `git push origin v0.1.0` — fires [release.yml](../.github/workflows/release.yml)
      which invokes the reusable [release-go.yml](../.github/workflows/release-go.yml)
      (goreleaser + cosign + syft SBOM + SLSA-L3 via `attest-build-provenance`).
- [ ] Verify release artifacts on the GitHub Releases page:
      multi-arch tarballs, `checksums.txt`, `checksums.txt.sig` + `.pem`,
      SPDX SBOM, `ghcr.io/golusoris/golusoris:v0.1.0` manifest, cosign
      signatures on each Docker digest.

## Risk register

| Risk | Mitigation |
| --- | --- |
| **API churn post-0.1.0** — breaking changes as real apps use the framework. | `Migration:` commit footer + apidiff gate + `golusoris bump` codemod pipeline. SemVer. |
| **Specialty sub-modules rot** (many `go.mod`s to keep green). | Renovate bumps them in lockstep; CI runs tests across all `go.mod`s in the repo. |
| **AI layer drift** — `.claude/skills/` + `AGENTS.md` fall out of sync. | Commit hook + CLAUDE.md convention: "every commit touching new/changed modules updates STATE.md + AGENTS.md". |
| **Security alerts accumulate** in specialty modules nobody imports. | Keep them in separate `go.mod` so alerts are scoped; document the policy in SECURITY.md. |

## Definition of "done" for v0.1.0

- Framework is usable by a net-new Go app via `go get github.com/golusoris/golusoris@v0.1.0`.
- Every module has an `AGENTS.md` explaining surface + assumptions.
- `examples/minimal` and `examples/full` build + run against the
  tagged version without `replace` directives.
- All CI gates green on the tagged SHA.

All four conditions hold as of commit `a979bd4` on `main`.
